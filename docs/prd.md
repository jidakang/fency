# PRD: 凿壁

## 1. 产品概述
### 1.1 文档标题和版本
   - PRD: 凿壁
   - 版本: 1.0

### 1.2 产品摘要
   - 凿壁是一个专注于健康医疗行业的AI解决方案平台。该平台旨在通过提供专业、易读、视觉吸引力强的内容，破解AI在医疗领域应用的障碍，同时也帮助AI行业更好地理解和进入医疗领域。
   - 平台内容主要围绕"信息化升级"、"科研辅助"、"AI技术与生态"和"知识报告"四大类别展开，旨在为医疗专业人士、AI从业者及对此领域感兴趣的用户提供有价值的信息和洞见。
   - 该项目强调现代化的设计、卓越的用户体验（包括响应式设计和深色/浅色模式切换）、内容的专业性与准确性，以及遵循最新的Web开发最佳实践。

## 2. 目标
### 2.1 业务目标
   - 建立"凿壁"作为健康医疗AI领域的权威信息来源和思想领导者品牌。
   - 通过高质量内容吸引并留住目标用户群体（医疗专业人士、AI从业者等）。
   - 展示在医疗AI领域的专业知识和解决方案能力。
   - (潜在) 通过"联系我们"功能促进潜在的合作或咨询机会。

### 2.2 用户目标
   - 方便地获取关于健康医疗领域AI应用和发展的专业、准确、易于理解的信息。
   - 通过清晰的分类和导航快速找到感兴趣的内容或特定解决方案。
   - 在各种设备上（桌面、平板、手机）都能获得舒适、美观的阅读体验。
   - 了解最新的AI技术（如大模型、多模态）及其在医疗场景的应用潜力。

### 2.3 非目标
   - 该平台不提供直接的医疗诊断或治疗建议。
   - 不作为用户生成内容的平台（如论坛或博客评论区）。
   - 不追求成为一个覆盖所有AI领域的综合性新闻网站。
   - 不包含复杂的后台用户管理或个性化功能（当前阶段）。

   - 医疗专业人士（医生、医院管理者、研究人员）
   - AI技术专业人士（开发者、研究员、产品经理）
   - 对AI+医疗领域感兴趣的学生或公众
   - （潜在）寻求合作的机构或企业代表

### 3.2 基本用户画像详情

   - **李工程师 (AI技术专业人士)**: AI公司的算法工程师，对将AI技术应用于医疗领域充满兴趣，希望了解医疗行业的痛点、数据特点以及AI落地的挑战。

## 4. 功能需求
   - **内容分类展示** (优先级: 高)
     - 网站需清晰展示"信息化升级"、"科研辅助"、"AI技术与生态"、"知识报告"四大内容分类。
     - 用户可以通过导航栏或页面上的分类标签切换不同分类下的文章列表。
   - **文章阅读** (优先级: 高)
     - 提供清晰、易读的文章展示页面。
     - 文章页面需包含标题、正文、作者信息（根据规则）、发布日期和最后修改日期。
   - **响应式设计** (优先级: 高)
     - 网站界面需在所有常见设备（手机、平板、桌面电脑）上都能完美适配和展示。
   - **深色/浅色模式切换** (优先级: 高)
     - 提供手动切换深色/浅色模式的功能。
     - 默认模式应能根据用户操作系统的偏好设置自动选择。
     - 用户的选择应能被本地存储，以便下次访问时保持一致。
   - **视觉与交互设计** (优先级: 高)
     - 整体界面需遵循规则中定义的"精致、专业、高级感"的设计风格。
     - 应用规则中更新后的配色方案 (#1a3a6d, #2a9d8f, #e09f3e)。
     - 使用指定的字体（Noto Sans SC, Noto Serif SC）和Font Awesome图标。
     - 包含规则中建议的视觉元素，如卡片、负空间、微交互等。
   - **日期显示** (优先级: 高)
     - 索引页的文章卡片需显示创建日期，并在适用时（30天内修改）显示更新日期。
     - 文章内容页面需在底部（作者信息前）显示创建日期和最后修改日期。
   - **返回首页链接** (优先级: 高)
     - 所有非首页的内容页面都需要有一个清晰可见的返回首页 (`index.html`) 的链接，通常在导航栏。
   - **作者与版权信息** (优先级: 高)
     - 页面底部需包含规则中指定的作者姓名、Email、头衔、微信公众号和版权信息。
     - 不得包含规则中禁止的其他联系方式（如地址、电话等）。
   - **数据可视化 (Mermaid)** (优先级: 中)
     - (如果文章内容需要) 支持使用Mermaid.js渲染图表（如流程图、关系图）以可视化复杂概念。
   - **内容搜索** (优先级: 低)
     - (未来迭代考虑) 提供关键词搜索功能，帮助用户快速查找相关文章。

## 5. 用户体验
### 5.1. 入口点和首次用户流程
   - 主要入口点为网站首页 (`index.html`)。
   - 首次用户流程：用户访问首页 -> 被Hero区域吸引，了解平台主旨 -> 浏览"关于我们"或直接查看内容分类 -> 点击感兴趣的分类标签 -> 浏览该分类下的文章卡片 -> 点击文章卡片进入阅读页 -> 阅读文章 -> (可选)切换深色/浅色模式 -> (可选)通过导航栏返回首页或探索其他分类。

### 5.2. 核心体验
   - **浏览分类:** 用户通过首页的分类导航标签（或导航栏菜单）清晰地选择感兴趣的内容主题。
     - 分类切换流畅，活动标签有明显视觉反馈。
   - **查阅文章列表:** 所选分类下的文章以卡片形式展示，包含标题、摘要、分类、日期等关键信息。
     - 卡片布局整洁，视觉吸引力强，加载快速。支持按日期倒序排列。
   - **阅读文章:** 文章页面排版清晰，字体易读，代码块（如果存在）格式化良好。
     - 阅读体验舒适，无干扰元素。页面底部包含日期和作者信息，并提供返回首页的途径。
   - **模式切换:** 用户可通过界面右上角的图标轻松切换深色和浅色模式。
     - 切换过程平滑，界面元素颜色随之正确变化。

### 5.3. 高级功能和边缘情况
   - 高级功能：深色/浅色模式切换、Mermaid图表展示（若有内容使用）。
   - 边缘情况：
     - 网络连接慢或中断时的页面加载情况（应有基本提示或优雅降级）。
     - 访问不存在的文章链接（应显示404页面或友好提示）。
     - 移动设备上的菜单交互（确保汉堡菜单易于点击和使用）。
     - Mermaid图表渲染失败（应有错误提示或不显示损坏的图表）。

### 5.4. UI/UX 亮点
   - 采用深海军蓝、柔和青色、赭黄色构成的专业且现代的配色方案。
   - 使用 Noto Serif SC 和 Noto Sans SC 字体组合，优化中文阅读体验。
   - 精致的卡片式设计、恰当的留白和视觉层次。
   - 流畅的分类切换动画和按钮悬停等微交互效果。
   - 提供沉浸式阅读体验的深色模式。
   - (潜在) 使用Mermaid图表进行信息可视化。

张医生是一名心血管内科医生，他希望了解最新的AI技术如何应用于心脏病风险预测和影像分析，以提升科室的诊疗水平。他在网上搜索相关信息时发现了"凿壁"平台。他被网站专业、现代的设计所吸引，并通过清晰的"科研辅助"和"AI技术与生态"分类，快速找到了几篇关于多模态AI在医疗影像分析和基于大模型的临床决策支持系统的深度报告。文章内容翔实、排版易读，还通过Mermaid图表直观展示了技术流程，让他对这些前沿技术有了更清晰的认识，并找到了可以进一步研究的方向。他对这个平台提供的专业内容和良好阅读体验感到满意。

## 8. 技术考量
### 8.1. 集成点
   - CDN 服务：用于加载 Tailwind CSS, Font Awesome, Noto Fonts, Mermaid.js。
   - （无其他明确的外部系统集成需求）

### 8.2. 数据存储与隐私
   - 主要内容为静态HTML文件，无需数据库存储用户数据或文章内容。
   - 主题偏好设置通过浏览器 `localStorage` 存储在客户端。
   - 遵守标准的Web隐私实践，不收集非必要的个人身份信息。

### 8.3. 可扩展性与性能
   - 静态网站架构具有良好的读取性能和高并发承载能力。
   - 遵循规则中的性能优化措施：图片压缩、代码最小化、惰性加载（若适用）。
   - 未来如需扩展动态功能（如搜索、用户评论），可能需要引入后端服务和数据库。

### 8.4. 潜在挑战
   - 保持内容的时效性和专业准确性，尤其是在快速发展的AI领域。
   - 确保所有Mermaid图表在不同浏览器和设备上都能正确渲染。
   - 长期维护静态链接的有效性，避免死链。
   - 如何在不牺牲设计感的前提下，持续优化页面加载性能。

## 9. 里程碑与计划
### 9.1. 项目估算
   - 中等: 2-4 周 (假设内容已基本准备好，主要为前端开发和集成)

### 9.2. 团队规模与构成
   - 小型团队: 1-2 人
     - 1 名前端工程师 (负责HTML/CSS/JS实现、响应式、主题切换)
     - 1 名内容编辑/产品经理 (负责内容整理、规则制定、验收)

### 9.3. 建议阶段
   - **阶段 1:** 基础框架与首页搭建 (1 周)
     - 关键交付物: 建立项目结构，集成Tailwind/FA/Fonts，完成首页 (`index.html`) 的HTML结构和基础样式，实现导航栏、Hero区域、关于我们、联系方式、页脚，实现基本的深色/浅色模式CSS变量。
   - **阶段 2:** 分类与文章展示 (1-2 周)
     - 关键交付物: 实现首页文章分类标签切换逻辑与样式，创建文章页面模板，实现文章内容展示、日期显示、作者信息显示，完成深色/浅色模式切换的JavaScript逻辑和LocalStorage持久化，确保全站响应式布局。
   - **阶段 3:** 细节优化与测试 (1 周)
     - 关键交付物: 添加微交互效果，集成Mermaid.js（如有需要），进行跨浏览器和设备测试，性能优化（图片、代码），修复发现的Bug，最终内容检查和部署。
