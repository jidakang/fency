为什么需要知识库？

在AI辅助编程领域，"幻觉"(Hallucinations)是一个常见问题，指AI在缺乏明确上下文时产生不准确或无关代码。研究表明，非控制的AI编码环境中，这类错误发生率约为40-60%。知识库技术提供了一种系统方法，通过六类详细文档为AI提供清晰上下文，可将错误率降至7-9%左右。

知识库技术概览

知识库技术由X用户阿西_出海(@axichuhai)于2025年3月24日提出（翻译自@PrajwalTomar_的英文帖子），旨在通过系统性文档减少AI编码工具如Cursor中的"幻觉"问题。该方法的核心理念是：

AI辅助编程的黄金法则

  • 人类负责规划：代码架构设计是人类任务
  • AI负责执行：编码实现是AI的强项
  • 详细文档是关键：通过六类文档为AI提供明确上下文

该方法特别适用于使用Cursor (v0.47)或Windsurf等AI编码助手的开发者，可通过工具如@CodeGuidedev自动生成所需文档，极大提升开发效率。

知识库六步法

项目需求文档 (PRD)

PRD是知识库的第一步，为整个项目定下基调。这份文档帮助AI对项目有清晰大框架认识，减少假设错误。

必须包含内容:

  • 应用概览与目标
  • 用户使用流程
  • 技术栈和API选择
  • 核心功能列表
  • 项目范围及非范围事项

精心设计的PRD能让AI充分理解项目目标和边界，避免偏离核心需求。

PRD 示例片段: Brandloop 项目

应用概览:

Brandloop是一款AI辅助日记和生产力应用，帮助用户记录日常见解并获取智能总结。

核心功能:
  • 语音转文本日记录入
  • AI日记内容自动总结
  • 第三方服务集成 (Google Calendar)
  • 数据分析与可视化
非项目范围:
  • 原生移动应用支持
  • 实时协作功能
  • 离线工作模式

应用流程文档

流程文档越清晰，AI表现越好。这份文档详细描述用户在应用中的导航路径与交互方式，提供具体细节，避免模糊表述。

核心要点:

  • 详细描述每个页面功能与用途
  • 明确用户从一个页面到另一个页面的跳转路径
  • 使用简洁直观的语言，突出关键细节
  • 避免模糊表述，具体说明交互逻辑

良好的流程文档能帮助AI理解应用的整体结构与用户体验设计意图。

应用流程文档示例: Braindump

登录页流程:

用户访问主页后看到登录/注册选项。点击"使用Google登录"按钮后，弹出授权对话框。授权成功后直接重定向至仪表盘页面。

仪表盘页面:

左侧导航栏显示："日记"、"洞察"、"设置"三个主要选项。中央区域默认显示最近7天的日记条目摘要。右上角提供"新建日记"按钮，点击后跳转到编辑器页面。

编辑器页面:

顶部工具栏包含格式选项和语音录入按钮。点击语音按钮开始录音，再次点击停止并转录。底部提供"保存"和"生成摘要"按钮，保存成功后提示用户并返回仪表盘。

技术栈文档

技术栈文档明确告诉AI使用哪些技术来开发项目，防止AI随意选择不适合的技术方案，确保开发环境的一致性。

应包含内容:

  • 前端框架与库（版本号）
  • 后端技术与服务
  • 数据库选择与ORM
  • 第三方API与服务（含链接）
  • 首选工具与配置方案

准确的技术栈文档能帮助AI使用最适合项目的技术组合，避免版本冲突和兼容性问题。

技术栈文档示例

前端技术:
  • React 18.2.0
  • Next.js 14.0.3
  • Tailwind CSS 3.3.0
  • Shadcn/UI 组件库
  • React Query v4 用于数据获取
后端服务:
  • Supabase （认证与数据库）
  • Next.js API Routes
  • OpenAI API （内容生成）
第三方服务:
  • Deepgram API (语音转文本)
  • Google OAuth (用户认证)
  • Stripe (支付处理)

前端指南

前端指南详细说明设计规范和UI要求，确保AI生成的界面美观一致，符合品牌调性，并提供良好的用户体验。

关键元素:

  • 字体系统（标题字体、正文字体、字号范围）
  • 配色方案（主色、辅助色、强调色、状态色）
  • 间距与布局规则
  • UI组件库与设计系统
  • 图标集与品牌资源

精细的前端指南能确保AI生成的界面保持视觉一致性，减少设计调整工作。

后端结构文档

后端结构文档明确定义数据存储和处理逻辑，特别适用于使用Supabase或Firebase等服务的项目，帮助AI理解数据关系和业务规则。

必要组成:

  • 数据库表结构与关系
  • API端点设计与功能
  • 认证与权限管理逻辑
  • 数据存储与检索规则
  • 边界情况与错误处理策略

完善的后端文档能帮助AI创建符合业务逻辑的数据处理系统，减少架构调整。

后端结构文档示例

数据库表结构:
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  email TEXT UNIQUE NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  display_name TEXT
);

CREATE TABLE journal_entries (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  content TEXT NOT NULL,
  summary TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
认证逻辑:

使用Google OAuth认证，通过Supabase Auth服务管理。用户首次登录时在users表中创建记录。访问所有API端点需要有效的JWT令牌。

存储规则:

语音录音临时存储在Supabase Storage的'audio'桶中，转录后删除。每个用户数据使用RLS（行级安全）限制，只能访问自己的内容。

实施计划

实施计划是对抗AI"幻觉"的杀手锏，列出50+具体编码步骤，将复杂项目分解为线性任务，让AI专注执行而非猜测。

计划特点:

  • 详细的步骤分解，从环境设置到功能实现
  • 清晰的执行顺序，避免开发冲突
  • 具体的技术实现指南，减少AI选择负担
  • 明确的检验点，确保功能正确实现

可使用ChatGPT生成初步计划，或通过@CodeGuidedev自动生成更精细的实施方案。

实施计划示例片段

  1. 环境设置: 创建Next.js 14项目，配置TypeScript和ESLint
  2. 安装依赖: 添加Tailwind CSS, Shadcn/UI, Supabase客户端
  3. 配置环境变量: 设置Supabase URL, API密钥和其他服务配置
  4. 身份验证设置: 实现Google OAuth登录流程
  5. 数据库迁移: 创建用户和日记条目表
  6. 布局组件: 创建主布局组件，包含导航和主题切换
  7. 登录页面: 实现登录页UI和认证流程
  8. 仪表盘页面: 创建用户仪表盘显示最近日记
  9. 日记编辑器: 实现富文本编辑器功能
  10. 语音录入: 集成Deepgram API实现语音转文本
  11. ...

（完整计划通常包含50+详细步骤）
